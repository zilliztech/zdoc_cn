# 组织与项目

Zilliz Cloud 通过组织和项目实现访问控制和资源逻辑分离。使用组织与项目功能，您可更精细地管理跨项目集群。

## 组织 {#organization}

组织是 Zilliz Cloud 中最高层级，组织中包含项目和成员。在大多数情况下，您可以根据公司或者其他分组来设置组织。

在组织下创建项目有助于高效分组和管理集群资源。账号所有者及组织管理员可以创建组织、邀请其他用户加入组织、为组织成员分配不同角色并授予不同级别的权限。

组织功能适用于在 Zilliz Cloud 上具有多个项目且需要不同用户见协作的场景。

## 项目 {#project}

项目是组织层级下的概念，用于将集群及相关资源进行分组。创建项目后，您可轻松管理资源，制定项目访问控制策略。

组织内每个项目的的数据都相互隔离。在大多数情况下，您可以根据公司中的部门或者团队来设置项目。每个项目内，可创建 1 个或多个集群。查询或搜索操作仅针对特定项目。

通过组织和项目，您可以精细化管理资源及实施资源访问控制。例如，账号所有者可以创建 1 个组织并邀请用户加入。然后，组织内所有被分配“组织管理员”角色的用户均可创建不同的项目并邀请项目成员。这样一来，某些用户只能访问被授予权限的资源，而无法访问无权限的资源。
