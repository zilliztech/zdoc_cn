删除 Entity

从 Collection 中删除一条或多条数据。

POST
https://{public_endpoint}/v1/vector/delete

示例​

说明

此处请使用由冒号（:）连接的集群用户名和密码做为 Token，如 user:password。

  • 删除一个 ID 为整数的 Entity.
curl --request POST \
     --url "${CLUSTER_ENDPOINT}/v1/vector/delete" \
     --header "Authorization: Bearer ${TOKEN}" \
     --header "accept: application/json" \
     --header "content-type: application/json" \
     -d '{
       "collectionName": "medium_articles",
       "id": 1
     }'
  • 删除一个 ID 为字符串的 Entity.
curl --request POST \
     --url "${CLUSTER_ENDPOINT}/v1/vector/delete" \
     --header "Authorization: Bearer ${TOKEN}" \
     --header "accept: application/json" \
     --header "content-type: application/json" \
     -d '{
       "collectionName": "medium_articles",
       "id": "id1"
     }'
  • 删除一组 ID 为整数的 Entity.
curl --request POST \
     --url "${CLUSTER_ENDPOINT}/v1/vector/delete" \
     --header "Authorization: Bearer ${TOKEN}" \
     --header "accept: application/json" \
     --header "content-type: application/json" \
     -d '{
        "collectionName": "medium_articles",
        "id": [1,2,3,4]
      }'
  • 删除一组 ID 为字符串的 Entity.
curl --request POST \
     --url "${CLUSTER_ENDPOINT}/v1/vector/delete" \
     --header "Authorization: Bearer ${TOKEN}" \
     --header "accept: application/json" \
     --header "content-type: application/json" \
     -d '{
        "collectionName": "medium_articles",
        "id": ["id1", "id2", "id3","id4"]
      }'

请求​

参数​

  • 无查询参数。

  • 路径参数

    参数名称参数说明
    public-endpointstring（必选）
    目标集群的 Endpoint。

请求体​

{
    "collectionName": "string",
    "id": "string"
}
参数名称参数描述
collectionNamestring（必选）
目标 Collection 名称。
idstring（必选）
目标 Entity ID。
{
    "collectionName": "string",
    "id": []
}
参数名称参数描述
collectionNamestring（必选）
目标 Collection 名称。
idarray（必选）
一组 Entity ID。
{
    "collectionName": "string",
    "id": "integer"
}
参数名称参数描述
collectionNamestring（必选）
目标 Collection 名称。
idinteger（必选）
目标 Entity ID。
{
    "collectionName": "string",
    "id": []
}
参数名称参数描述
collectionNamestring（必选）
目标 Collection 名称。
idarray（必选）
一组 Entity ID。

响应​

返回空对象。

响应体​

  • 处理请求成功后返回
{
    "code": 200,
    "data": {}
}
  • 处理请求失败后返回
{
    "code": integer,
    "message": string
}

属性​

下表罗列了响应包含的所有属性。

属性名称属性描述
codeinteger
表示请求是否成功。
  • 200：请求成功。
  • 其它：存在错误。
dataobject
表示响应中携带的数据对象。
messagestring
具体描述请示错误的原因。
