# 指标与告警快速参考

在本手册中，您可以找到 Zilliz Cloud 集群监控指标的描述，以及您可以在组织和项目级别设置的告警项。

## 可用指标{#available-metrics}

Zilliz Cloud 控制台中的**指标**页签展示了各类集群指标图表。下表描述了各种指标的含义以及在达到相应指标值时建议采取的相关操作。

|  指标名称                |  单位      |  描述                                                                                                                                                                                             |  用例                                                                                                                                                                                                                                                                                                                                                     |
| -------------------- | -------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
|  **资源类**             |          |                                                                                                                                                                                                 |                                                                                                                                                                                                                                                                                                                                                         |
|  CU 计算资源使用率<br/>  |  %       |  相对于 CU 总计算能力的已使用计算能力的度量。                                                                                                                                                                       |  监控 CU 计算资源的利用情况，以确定集群是否能够处理未来的工作负载。<br/> - 当 CU 计算资源使用率超过特定阈值时，[配置告警](./manage-project-alerts)以接收告警通知。<br/> - 当 CU 计算资源使用率达到 70%-80% 时，请检查您的服务状态，并准备扩容。<br/> - 当 CU 计算资源使用率超过 90% 时，请立即[扩容](./manage-cluster#scale-up-cluster)以避免任何服务中断。<br/> - 请注意，不同的 CU 类型具有不同的性能和容量。请确保您[选择最合适的 CU 类型](./cu-types-explained)。<br/>                  |
|  CU 中已加载容量使用率        |  %       |  相对于 CU 总容量的已使用容量的度量。                                                                                                                                                                           |  监控 CU 容量的利用情况，以确定集群是否能够处理未来的工作负载。<br/> - 当 CU 容量达到 70%-80% 时，请检查您的服务状态，并准备扩容。<br/> - 当 CU 容量超过 90% 时，您将收到紧急告警。请立即[扩容](./manage-cluster#scale-up-cluster)以避免任何服务中断。<br/> - 当 CU 容量达到 100% 时，您将无法向集群写入数据。请立即[扩容](./manage-cluster#scale-up-cluster)以避免服务中断。<br/> - 请注意，不同的 CU 类型具有不同的性能和容量。请确保您[选择最合适的 CU 类型](./cu-types-explained)。<br/> |
|  存储用量                |  GB      |  数据和索引消耗的持久存储总量。                                                                                                                                                                                |  监控集群中数据和索引所消耗的存储资源用量。<br/> - 当存储使用超过特定阈值时，[配置告警](./manage-project-alerts)以接收告警通知。<br/> - 当存储使用不足时，请考虑：<br/>     - 释放未使用的索引<br/>     - 释放未使用的 collection<br/>     - 扩展您的集群。请注意，不同的 CU 类型具有不同的性能和容量。请确保您[选择最合适的 CU 类型](./cu-types-explained)。<br/>                                                                                     |
|  **性能类**             |          |                                                                                                                                                                                                 |                                                                                                                                                                                                                                                                                                                                                         |
|  QPS                 |  QPS     |  每秒 search、query、批量插入和 upsert 请求的次数。<br/> - Search QPS 用紫色线表示。<br/> - Query QPS 用蓝色线表示。<br/> - 批量插入 QPS 用橙色线表示。<br/> - Upsert QPS 用绿色线表示。<br/>                                   |  监控各类操作的 QPS 以追踪系统性能。<br/> 要了解您的系统 QPS 水平，请参考[性能测试工具](https://zilliz.com.cn/vector-database-benchmark-tool)。                                                                                                                                                                                                                                         |
|  VPS                 |  VPS     |  在 search、query、批量插入和 upsert 请求中的向量操作次数。VPS = QPS * n（n 代表操作中涉及的向量数量）。<br/> - Search VPS 用紫色线表示。<br/> - Query VPS 用蓝色线表示。<br/> - 批量插入 VPS 用橙色线表示。<br/> - Upsert VPS 用绿色线表示。<br/> |  监控各类操作的 VPS 以追踪系统性能。                                                                                                                                                                                                                                                                                                                                   |
|  查询延迟                |  ms      |  客户端发送请求到服务器并收到响应所经过的时间。包括平均延时和 P99 延时。<br/> - Search 延时用紫色线表示。<br/> - Query 延时用蓝色线表示。<br/> - 批量插入延时用橙色线表示。<br/> - Upsert 延时用绿色线表示。<br/>                                         |  监控操作延时以追踪系统效率。<br/> - 要了解您的系统延时水平，请参考[性能测试工具](https://zilliz.com.cn/vector-database-benchmark-tool)。<br/> - 在大多数情况下，搜索请求会在毫秒内得到响应。如果响应时间延长到秒或更长，可能表明集群过载。此外，当单个请求中批量操作的数量过多时，也可能出现较长的延时。<br/>                                                                                                                                                 |
|  **业务类**             |          |                                                                                                                                                                                                 |                                                                                                                                                                                                                                                                                                                                                         |
|  请求成功率               |  %       |  每秒所有请求中成功请求的百分比。                                                                                                                                                                               |  监控请求成功率。<br/> 当请求成功率超过特定阈值时，[配置告警](./manage-project-alerts)以接收告警通知。                                                                                                                                                                                                                                                                                 |
|  请求失败率               |  %       |  每秒所有请求中超时请求的百分比。<br/>                                                                                                                                                                       |  监控请求失败率。<br/> 当请求失败率超过特定阈值时，[配置告警](./manage-project-alerts)以接收告警通知。                                                                                                                                                                                                                                                                                 |
|  已加载 Entity 数        |  count/s |  集群中已加载的 entity 数量。                                                                                                                                                                             |  监控集群中的 entity 数量。<br/> 当加载的 entity 超过特定阈值时，[配置告警](./manage-project-alerts)以接收告警通知。                                                                                                                                                                                                                                                                  |
|  Entity 插入速率         |  count/s |  每秒插入的 entity 数量。                                                                                                                                                                               |  监控 entity 插入的效率。<br/> 当 entity 插入率下降时，请考虑扩容集群或调整请求率。                                                                                                                                                                                                                                                                                                |

## 组织告警{#organization-alerts}

组织告警用于提醒您账单相关的重要信息，如信用卡到期提醒、免费额度使用情况、现金余额警示以及用量金额相关的告警通知。

|  告警项             |  单位             |  告警等级                     |  默认告警条件                                                                    |  默认状态        |
| ---------------- | --------------- | ------------------------- | -------------------------------------------------------------------------- | ------------ |
|  信用卡有效期<br/>  |  天              |  **告警** / **紧急**<br/>  |  - **告警**：信用卡有效期小于 30 天时触发告警。<br/> - **紧急**：信用卡有效期小于 7 天时触发告警。<br/>  |  开启          |
|  免费额度（余额/有效期）    |  % / 天<br/>  |  **告警**                   |  免费额度余额小于 ¥10 或免费额度有效期小于 0 天时触发告警。                                         |  开启<br/>  |
|  现金余额            |  ¥              |  **紧急**                   |  现金余额小于 ¥100 时触发告警。                                                        |  开启          |
|  用量金额            |  ¥              |  **告警**                   |  用量金额大于 ¥100 时触发告警。                                                        |  关闭          |

## 项目告警{#project-alerts}

项目告警主要关注您集群的性能方面，包括 CU 使用情况、QPS 阈值、延时问题以及请求异常等，确保您能够维持最佳的集群性能。

|  告警项                |  单位  |  告警等级        |  默认告警条件                                                                                                    |  默认状态        |
| ------------------- | ---- | ------------ | ---------------------------------------------------------------------------------------------------------- | ------------ |
|  CU 计算资源<br/>    |  %   |  **告警 / 紧急** |  - **告警**：CU 计算资源用量大于 70% 且持续时间超过 10 分钟时触发告警。<br/> - **紧急**：CU 计算资源用量大于 90% 且持续时间超过 10 分钟时触发告警。<br/> |  开启<br/>  |
|  CU 加载容量            |  %   |  **告警 / 紧急** |  - **告警：**CU 加载容量大于 70% 且持续时间超过 10 分钟时触发告警。<br/> - **紧急：**CU 加载容量大于 90% 且持续时间超过 10 分钟时触发告警。<br/>     |  开启          |
|  Search 类型请求数       |  QPS |  **告警**      |  Search 类型请求数大于 50 QPS 且持续时间超过 10 分钟时触发告警。                                                                 |  关闭          |
|  Query 类型请求数        |  QPS |  **告警**      |  Query 类型请求数大于 50 QPS 且持续时间超过 10 分钟时触发告警。                                                                  |  关闭          |
|  Search 类型 P99 请求延时 |  ms  |  **告警**      |  Search 类型 P99 请求延时大于 1000 ms 且持续时间超过 10 分钟时触发告警。                                                          |  关闭          |
|  Query P99 请求延时     |  ms  |  **告警**      |  Query 类型 P99 请求延时大于 1000 ms 且持续时间超过 10 分钟时触发告警。                                                           |  关闭          |
|  请求失败率              |  %   |  **告警**      |  请求失败率大于 50 % 且持续时间超过 10 分钟时触发告警。                                                                          |  开启          |
